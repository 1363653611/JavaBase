### 等待通知 ### 
*   经典模式:

-  等待方原则：
 1. 获取对象的锁。
 2. 如果条件不满足，那么调用对象的wait()方法，被通知后仍要检查条件。
 3. 条件满足则执行对应的逻辑
 
```java
	synchronized(对象) {
		while(条件不满足) {
			对象.wait();
		}
		对应的处理逻辑
	}
```

- 通知方原则
----
	1. 获得对象的锁。
	2. 改变条件。
	3. 通知所有等待在对象上的线程。

伪代码：

```java
	synchronized(对象) {
		改变条件
		对象.notifyAll();
	}
```
